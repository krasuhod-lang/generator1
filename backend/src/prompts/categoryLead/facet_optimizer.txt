КОНТЕКСТ
Мы оптимизируем категорию интернет-магазина {{CATEGORY}}. Есть текущая структура
фильтров (названия групп и значения чекбоксов), созданная без учёта SEO, и список
реальных поисковых запросов пользователей. Цель — превратить фильтры в точки
входа для органического трафика (long-tail SEO): страницы фильтрации должны стать
полноценными посадочными страницами, релевантными запросам.

ЗАДАЧА
Проведи глубокий анализ соответствия текущей структуры фильтров поисковому спросу
(search intent) и выполни шаги:

1. СЕМАНТИЧЕСКАЯ НОРМАЛИЗАЦИЯ (Rename). Сравни текущие названия фильтров и
   значений с лексикой пользователей в семантическом ядре. Предложи переименование
   групп и значений так, чтобы они совпадали с формулировками запросов.
   Пример: «Тип: Для игр» → «Назначение: Игровые».
2. АНАЛИЗ УПУЩЕННЫХ ВОЗМОЖНОСТЕЙ (Gap / New). Найди кластеры запросов, для которых
   сейчас нет фильтра. Предложи создание новых фильтров (свойств).
3. КОНСОЛИДАЦИЯ И ЧИСТКА (Merge / Delete). Выяви избыточные значения, дробящие вес
   страницы и не имеющие спроса (синонимы, слишком узкие технические параметры).
   Предложи объединение там, где это логично.
4. ПРИОРИТЕЗАЦИЯ ИНДЕКСАЦИИ. Отметь, какие фильтры/комбинации имеют наибольший
   SEO-потенциал (High Volume / Low Competition) и должны формировать ЧПУ-URL, H1,
   Meta и быть открыты для индексации; что — закрыть в noindex.

ФОРМАТ ВЫВОДА
Ответь СТРОГО валидным JSON без markdown-обёрток (без ```json), без пояснений.
Ответ начинается с { и заканчивается }. Используй одинарные кавычки (') внутри
строк. Структура:
{
  "rows": [
    {
      "current": "текущий фильтр или значение",
      "seo_name": "предлагаемое SEO-название",
      "action": "Rename | New | Merge | Delete",
      "reason": "обоснование на основе запросов",
      "index_priority": "High | Med | Low"
    }
  ],
  "top_recommendations": ["рекомендация 1", "рекомендация 2", "рекомендация 3"],
  "noindex_list": ["мусорный фильтр/значение, который лучше закрыть от индексации"]
}
Колонка "action" — одно из: Rename, New, Merge, Delete. Колонка "index_priority" —
одно из: High, Med, Low. Не добавляй другие значения.
