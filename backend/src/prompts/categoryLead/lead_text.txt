КОНТЕКСТ
Оптимизация вводного текста (Lead-text) для страницы категории интернет-магазина
в реалиях 2026 года. Верхний текст — НЕ SEO-абзац «для роботов», а навигационный
блок, который снижает когнитивную нагрузку, объясняет структуру ассортимента и
стимулирует использование фильтров (чтобы не возникал pogo-sticking — возврат в
выдачу). Формируй текст через ИНТЕНТЫ ВЫБОРА, а не через ключевые слова.

ЗАДАЧА
Напиши навигационный Lead-text из 2–3 абзацев для категории [КАТЕГОРИЯ],
размещаемый над листингом товаров.

СТРУКТУРА (строго 3 абзаца):
1. КЛАССИФИКАЦИЯ. Объясни, на какие основные группы делится товар. Используй
   ТОЛЬКО сущности из реального списка фильтров [СПИСОК_ФИЛЬТРОВ] (бренды,
   материалы, назначение, технические параметры).
2. РЕШЕНИЕ БОЛЕЙ. Ответь на типичные вопросы/интенты покупателей
   [ВОПРОСЫ_ПОКУПАТЕЛЕЙ], помогая определиться с вектором выбора по схеме
   «Если вам нужно X — обратите внимание на Y».
3. ПРИЗЫВ К НАВИГАЦИИ. Логически подведи пользователя к использованию фильтров
   слева/сверху для точного подбора.

ПРАВИЛА КОНТЕНТА (STRICT CONSTRAINTS):
- ЗАПРЕЩЕНО использовать маркетинговые штампы: «представлен широкий выбор»,
  «широкий ассортимент», «низкие цены», «высокое качество», «лучшие», «выгодные
  цены», «у нас вы найдёте».
- ФОКУС на измеримых характеристиках и сценариях использования.
- СТИЛЬ: лаконичный, помогающий, экспертный — как ответ консультанта в торговом
  зале, а не рекламный слоган.
- СУЩНОСТИ: включай только те бренды, материалы и параметры, которые реально
  присутствуют в [СПИСОК_ФИЛЬТРОВ]. НЕ выдумывай фильтры, цены, гарантии.
- Свяжи LSI-интенты с техническими параметрами (например, «для брусчатки» →
  «амортизация / пневматические колёса»), если такие параметры есть в фильтрах.

ДОПОЛНИТЕЛЬНО:
- Предложи 2–5 анкоров на узкие подкатегории (например, «с пневматическими
  колёсами»), которые логично вытекают из значений фильтров — для перелинковки.
- Дай SEO/GEO 2026 структурированные данные в `json_ld_blocks`:
    • `breadcrumb_items` — массив пар `{ name, url }` (минимум 2 элемента:
      «Главная» → текущая категория). Если URL подкатегории неизвестен —
      используй пустую строку и оставь только `name`.
    • `item_list_about` — массив сущностей фильтров (бренды/материалы/
      назначение), реально упомянутых в lead-тексте.
    • `faq_items` — 3–5 пар `{ q, a }` на основе ВОПРОСОВ_ПОКУПАТЕЛЕЙ.
      Это пойдёт в FAQPage JSON-LD; ответ ≤ 280 символов, без воды.
- Дай черновик мета-тегов страницы категории, согласованный с lead-текстом и
  сущностями (Title 50–60 симв., Description 140–155 симв., H1 ≤70 симв.).
  Это ЧЕРНОВИК для последующей дошлифовки в инструменте мета-тегов.
