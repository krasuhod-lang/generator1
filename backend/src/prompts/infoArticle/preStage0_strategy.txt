INFO-ARTICLE PRE-STAGE 0 — Стратегический контекст темы информационной статьи в блог (DSPy-Signature).

[INPUTS]
  topic            : тема статьи (строка), обязательно.
  region           : регион / гео (строка), может быть пустой.
  brand_name       : бренд автора (строка, может быть пустой).
  brand_facts      : короткие факты о бренде/нише (строка, может быть пустой).
  commercial_domains : список доменов, на которые статья будет перелинковываться
                       (массив строк, ≤ 8 уникальных). Это контекстный сигнал ниши,
                       а НЕ список тем для упоминания.
  commercial_h1_sample : 5–10 примеров H1 коммерческих разделов (массив строк) —
                         нужны, чтобы понять тип бизнеса автора (услуги/товары/курсы).

[OUTPUT]
  Строгий JSON (без markdown, без пояснений снаружи):
  {
    "niche_summary":         "string",            // 1–2 предложения, что за ниша
    "article_type_hint":     "how-to|listicle|guide|comparison|explainer|case-study",
    "inferred_brand_role":   "string",            // например «независимый консультант», «агентство», «производитель»
    "default_brand_name":    "string",            // если brand_name пустой — выведи разумный плейсхолдер из commercial_domains
    "default_author_persona":"string",            // короткое описание автора (1 фраза)
    "intent_types": [                              // 1–3 типа поискового интента, обслуживаемых статьёй
      "informational|commercial-investigation|problem-solving|preventative|navigational|transactional"
    ],
    "audience_clusters": [
      { "name": "string",
        "intent_bias": "informational|commercial|navigational|transactional",
        "buyer_journey_stage": "awareness|consideration|decision|retention",
        "pains": ["string"], "goals": ["string"] }
    ],
    "topical_opportunities": [
      { "angle": "string", "why_relevant": "string", "priority": "high|medium|low" }
    ],
    "lsi_entities": [                              // обязательные сущности/термины ниши, без которых тема не раскрыта
      { "entity": "string",
        "category": "term|method|tool|metric|standard|organization|product|symptom|condition|other",
        "must_appear": true,
        "short_definition": "string" }
    ],
    "brand_integration_hooks": [                   // как органично связать БОЛЬ ЦА с конкретным ФАКТОМ бренда
      { "audience_pain": "string",
        "brand_fact": "string",                    // дословный/перефразированный факт из brand_facts (ничего не выдумывать)
        "hook_idea": "string",                     // как именно вписать в текст (1–2 предложения)
        "section_hint": "intro|problem|solution|comparison|process|cta|faq|other" }
    ],
    "demand_signals": [
      { "query_class": "how-to|definition|comparison|list|review|other",
        "sample_queries": ["string"], "user_pain": "string" }
    ]
  }

[REASONING STEPS — silent, do not output]
  1. Из commercial_h1_sample и commercial_domains выведи область деятельности
     автора и его tone-of-voice (B2B/B2C, премиум/масс, услуги/товары).
  2. Свяжи `topic` с этой нишей. Определи article_type_hint: что именно
     лучше написать (how-to, листикл, гайд, сравнение, объяснение).
  3. Определи intent_types: какой именно поисковый интент закрывает статья.
       • informational — пользователь хочет ПОНЯТЬ.
       • commercial-investigation — пользователь СРАВНИВАЕТ варианты решения.
       • problem-solving / diagnostic — у пользователя уже проблема, ищет шаги решения и чеклист.
       • preventative — пользователь хочет избежать проблемы / снизить риски.
       • navigational / transactional — добавляй только если статья реально подталкивает к действию.
       Выбирай 1–3 наиболее точных. Это диктует структуру: для problem-solving обязателен чеклист анализов / диагностики и методы решения; для commercial-investigation — таблица сравнения; для preventative — превентивные меры и регулярные действия.
  4. Если region задан — учитывай локальную специфику (право, климат, привычки).
  5. Для default_brand_name: если brand_name пустой, возьми чистый домен
     (без http/www) первого commercial_domains, из него получи разумную
     торговую марку (Capitalize, без TLD).
  6. Сформулируй 2–4 аудиторных кластера с конкретными pains/goals и
     обязательным buyer_journey_stage.
  7. Подбери 3–6 сильных углов раскрытия темы; все на 100 % внутри `topic`.
  8. Сгенерируй lsi_entities — 8–15 обязательных сущностей/терминов ниши
     (для медицинской ниши пример: «спермограмма», «морфология по Крюгеру»,
     «АМГ», «овариальный резерв», «микро-ТЕЗЕ», «витрификация», «ХГЧ»;
     для авто-ниши: «эндоскопия цилиндров», «контрактный мотор», «тотал»,
     «запрет на рег. действия»). Для каждого: категория, краткое определение,
     must_appear=true для тех, без которых тема не раскрыта (минимум 5 must).
  9. Сгенерируй brand_integration_hooks — для каждой ключевой боли ЦА
     найди в `brand_facts` подходящий ФАКТ (только реальный, без выдумок!)
     и предложи hook_idea: как органично вписать этот факт в текст,
     не превращая статью в рекламу. Пример: «боязнь потери клеток при
     разморозке» → факт «у клиники свой криобанк с витрификацией Cryotop»
     → hook «в разделе про этапы криоконсервации упомянуть, что
     современные клиники используют витрификацию Cryotop, обеспечивающую
     >95% выживаемости клеток». Если brand_facts пуст — массив пуст.
 10. Сгенерируй 3–6 кластеров демандов — типичные поисковые запросы и боль.
 11. Не выдумывай статистику, цифры, имена — это стратегический анализ.

[OUTPUT CONTRACT]
  - Только валидный JSON, без markdown-обёрток, без комментариев.
  - Все массивы могут быть пустыми, но ключи обязательны.
  - Русский язык для содержательных полей. Enum-значения — английский.
  - Любые строки — не длиннее 280 символов.
  - В brand_integration_hooks запрещено выдумывать факты бренда —
    только то, что реально упомянуто в brand_facts на входе.

[SELF-AUDIT — silent, do not output]
  ✓ article_type_hint логически вытекает из topic + audience_clusters?
  ✓ intent_types согласован с article_type_hint и аудиторией?
  ✓ default_brand_name вменяемо производен от доменов?
  ✓ Нет выдуманной статистики, цитат, конкретных брендов помимо default_brand_name?
  ✓ Все angles/кластеры — на 100 % внутри `topic`?
  ✓ В lsi_entities — реальные термины ниши, не маркетинговые слова, минимум 5 must_appear?
  ✓ brand_integration_hooks опираются ТОЛЬКО на реальный brand_facts (если он пуст — массив пуст)?
  ✓ У каждого audience_cluster есть buyer_journey_stage?

[STRICT OUTPUT RULES]
  - Верни ТОЛЬКО JSON. Никаких префиксов/суффиксов, markdown-блоков.
  - Запрещены пустые niche_summary и audience_clusters.
  - Длина любой строки ≤ 280 символов.
