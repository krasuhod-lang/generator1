EXPAND_SECTION / Написать новый раздел статьи с нуля (DSPy-Signature).

[INPUTS]
  user_instruction    : тема нового раздела (часто формат «H2: ...»).
  full_article_text   : текст уже написанной статьи — нужен, чтобы НЕ дублировать.
  audience_profile    : целевая аудитория, её цели и возражения.
  eeat_brief          : E-E-A-T требования ниши (опыт, экспертиза, авторитет, доверие).
  lsi_unused          : LSI-слова, которые ещё НЕ покрыты в основном тексте.
  niche_terminology   : профессиональная лексика ниши.

[OUTPUT]
  new_section_html    : HTML-фрагмент нового раздела, готовый к вставке ниже текущего курсора.

[REASONING STEPS — выполни молча, в ответе НЕ выводи]
  1) Определи тему раздела: если user_instruction начинается с «H2:» — используй
     эту строку как заголовок дословно; иначе сформулируй сам, в стиле других H2 статьи.
  2) Просмотри full_article_text и выпиши мысленно темы, которые УЖЕ раскрыты —
     запретное множество, которое в новом разделе НЕ повторяешь.
  3) Спланируй структуру: 2–4 абзаца + минимум 1 список (<ul> / <ol>) или
     таблица (<table>). Каждый абзац — самостоятельная подмысль.
  4) Заложи ≥1 элемент E-E-A-T из eeat_brief: личный опыт, экспертный вывод,
     профессиональная терминология, кейс, цифра, чек-лист.
  5) Распредели ≥5 слов из lsi_unused органично по абзацам и спискам.
  6) Проверь, что в тексте НЕТ внешних/внутренних ссылок и нет «гарантий».

[OUTPUT CONTRACT — обязательные ограничения]
  - Начни с <h2>Заголовок раздела</h2>; разрешены вложенные <h3>, но не <h1>/<h4>+.
  - Чистого текста (без HTML-тегов): 350–700 слов.
  - 2–4 абзаца <p> + минимум 1 список (<ul>/<ol>) или таблица (<table>).
  - Используй ≥5 слов из lsi_unused и ≥1 термин из niche_terminology.
  - НЕ дублируй информацию, уже раскрытую в full_article_text.
  - ЗАПРЕЩЕНО: <a href=...>, <a>...<\/a>, <img>, <script>, <iframe>, <html>, <body>.
  - ЗАПРЕЩЕНО: «гарантируем», «100 % результат», «лучший в мире», «без преувеличений».
  - НЕ возвращай пустой ответ.

[SELF-AUDIT — пройди молча перед выводом]
  ✓ Раздел начинается с одного <h2>?
  ✓ 2–4 <p> + ≥1 <ul>/<ol>/<table>?
  ✓ ≥5 LSI и ≥1 термин ниши?
  ✓ Не дублирует темы из full_article_text?
  ✓ Нет ссылок, нет запрещённых формулировок?
  ✓ 350–700 слов чистого текста?
  Если хотя бы один пункт «нет» — переработай ответ ДО вывода.

[STRICT OUTPUT RULES]
  Выведи ТОЛЬКО HTML new_section_html — без обёрток <html>/<body>, без markdown-fence,
  без вводных. Никогда не возвращай пустой ответ.
