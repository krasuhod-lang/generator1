ANTI_SPAM / Снижение плотности повторяющегося ключевого слова (DSPy-Signature).

[INPUTS]
  selected_text       : HTML/Markdown — фрагмент с переспамом.
  keyword             : слово / фраза, плотность которой нужно снизить
                        (передаётся в [extra: keyword=...] либо в user_instruction).
  lsi_unused          : LSI-слова, которые можно использовать как синонимы.
  niche_terminology   : тематический словарь — источник перифраз.

[OUTPUT]
  rewritten_fragment  : HTML/Markdown — фрагмент с пониженной плотностью ключа.

[REASONING STEPS — выполни молча, в ответе НЕ выводи]
  1) Найди в selected_text ВСЕ упоминания keyword (включая склонения и падежи).
  2) Сосчитай их. Целевое количество в финале: 1–2 упоминания на весь фрагмент.
  3) Для лишних упоминаний выбери замену:
       a) семантический синоним из lsi_unused или niche_terminology;
       b) перифраза («услуга», «такой подход», «эта методика»);
       c) местоимение или анафора («он», «она», «такой инструмент»),
          если из контекста однозначно понятен референт.
  4) Согласуй окружающие слова в роде/числе/падеже.
  5) Не меняй смысл, цифры, факты, ссылки, заголовки H2/H3, списки и
     выделения <strong>/<em>.

[OUTPUT CONTRACT — обязательные ограничения]
  - keyword (в любых формах) встречается в rewritten_fragment ≤ 2 раз.
  - Каждая замена — осмысленный синоним / перифраза / местоимение.
    ЗАПРЕЩЕНО: оставлять «голую» дыру или вырезать вхождение без замены, если
    это ломает смысл предложения.
  - НЕ вводи новые «ключи», которых нет ни в lsi_unused, ни в исходной нише.
  - Сохрани все ссылки <a href="...">, выделения <strong>/<em>, заголовки H2/H3,
    списки <ul>/<ol> — структура должна остаться идентичной.
  - Длина ответа: 0.85 × len(selected_text) ≤ len(answer) ≤ 1.10 × len(selected_text).
  - НЕ возвращай пустой ответ.

[SELF-AUDIT — пройди молча перед выводом]
  ✓ Сколько раз keyword встречается в финале? (цель: ≤ 2)
  ✓ Каждая замена — осмысленный синоним / перифраза / местоимение?
  ✓ Сохранены все ссылки, выделения, заголовки, списки?
  ✓ Сохранён исходный смысл?
  ✓ Уложился ли в ±10–15 % по длине?
  Если хотя бы один пункт «нет» — переработай ответ ДО вывода.

[STRICT OUTPUT RULES]
  Выведи ТОЛЬКО rewritten_fragment — без обёрток ```html```, без префиксов,
  без пояснений. Никогда не возвращай пустой ответ.
