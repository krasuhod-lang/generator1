ADD_FAQ / Сгенерировать блок «Вопросы и ответы» (DSPy-Signature).

[INPUTS]
  full_article_text   : полный HTML статьи — нужен, чтобы НЕ дублировать темы.
  audience_profile    : целевая аудитория и её болевые точки.
  lsi_unused          : список LSI-слов, которые ещё НЕ покрыты в основном тексте.
  niche_terminology   : профессиональная лексика ниши.

[OUTPUT]
  faq_block           : валидный HTML — секция <section class="faq"> с ровно
                        4 парами «вопрос — ответ» в тегах <details>/<summary>/<p>.

[REASONING STEPS — выполни молча, в ответе НЕ выводи]
  1) Прочитай full_article_text и составь мысленный список тем, которые УЖЕ раскрыты.
  2) Сформулируй 4 РАЗНЫХ вопроса от лица целевой аудитории — это должны быть
     реальные поисковые интенты, а НЕ «соломенные» вопросы. Вопрос обязан
     закрывать пробел, которого в основном тексте НЕТ.
  3) Для каждого ответа подбери 2–4 предложения по существу: цифры, шаги,
     условия, оговорки, экспертный нюанс. Без воды и канцелярита.
  4) Распредели минимум 5 LSI-слов из lsi_unused органично между всеми 4
     парами (можно одно слово на пару). Используй грамматически уместную
     форму, склоняй по падежу/числу.
  5) Проверь, что вопросы НЕ повторяют формулировки заголовков статьи.

[OUTPUT CONTRACT — обязательный шаблон, без отклонений]
<section class="faq">
  <h2>Вопросы и ответы</h2>
  <details><summary>Вопрос 1?</summary><p>Ответ 1.</p></details>
  <details><summary>Вопрос 2?</summary><p>Ответ 2.</p></details>
  <details><summary>Вопрос 3?</summary><p>Ответ 3.</p></details>
  <details><summary>Вопрос 4?</summary><p>Ответ 4.</p></details>
</section>

  - Ровно 4 блока <details>; внутри каждого — один <summary> + один <p>.
  - Никаких других тегов, классов, обёрток. Никаких внешних ссылок (<a>).
  - Без вводных, без «Вот ваш FAQ», без markdown-fence (```html и т. п.).
  - Каждое «Вопрос N?» должно заканчиваться знаком «?». Каждое «Ответ N.» —
    точкой/восклицательным/вопросительным знаком.

[SELF-AUDIT — пройди молча перед выводом]
  ✓ Ровно 4 пары <details>?
  ✓ Все 4 вопроса разные и НЕ дублируют темы из full_article_text?
  ✓ ≥ 5 LSI-слов из lsi_unused использованы органично?
  ✓ Ответы 2–4 предложения, есть конкретика?
  ✓ HTML валидный, нет лишних тегов / атрибутов?
  Если хотя бы один пункт «нет» — переработай ответ ДО вывода.

[STRICT OUTPUT RULES]
  Выведи ТОЛЬКО HTML faq_block — без префиксов, без пояснений, без markdown-обёрток.
  Никогда не возвращай пустой ответ.
