CUSTOM / Свободный запрос пользователя (DSPy-Signature).

[INPUTS]
  selected_text       : HTML/Markdown — выделенный фрагмент ИЛИ маркер
                        «— (фрагмент не выделен)».
  user_instruction    : произвольная задача от пользователя.
  full_article_text   : контекст всей статьи.
  audience_profile    : целевая аудитория и её болевые точки.
  eeat_brief          : требования E-E-A-T для ниши.
  niche_terminology   : профессиональная лексика ниши.
  lsi_unused          : LSI-слова, ещё не покрытые в основном тексте.

[OUTPUT]
  result_html         : HTML-фрагмент, готовый к ВСТАВКЕ В РЕДАКТОР.

[REASONING STEPS — выполни молча, в ответе НЕ выводи]
  1) Доработай (расширь и уточни) user_instruction: если запрос короткий или
     неоднозначный — мысленно сформулируй его максимально полно с учётом
     E-E-A-T, тональности, ниши и аудитории.
  2) Определи режим работы по наличию selected_text:
       a) ЕСТЬ фрагмент → режим REPLACE: переписать ТОЛЬКО выделенный кусок.
       b) НЕТ фрагмента → режим INSERT: создать новый блок ниже курсора.
  3) Если фрагмент содержит грамматические, пунктуационные, фактические,
     стилистические или логические ошибки — обязательно устрани их в финальном
     тексте, даже если пользователь не просил этого явно.
  4) Если уместно — органично используй 2–3 LSI-слова из lsi_unused.

[OUTPUT CONTRACT — обязательные ограничения]
  Режим REPLACE (selected_text реальный):
    - Верни ИСКЛЮЧИТЕЛЬНО переписанный/исправленный вариант ТОЛЬКО этого
      выделенного фрагмента (он будет автоматически подставлен в редактор
      вместо выделения).
    - Длина: 0.85 × len(selected_text) ≤ len(answer) ≤ 1.15 × len(selected_text)
      (если задача явно не требует обратного).
    - Сохрани исходную HTML/Markdown-разметку фрагмента (теги <p>, <h2>, <h3>,
      <ul>, <li>, <strong>, <em>, <a> и т. д.). Не добавляй обёрток
      (например, <article>, <section>), которых не было.
    - Не добавляй и не удаляй H2/H3, если задача об этом не просит.
    - Сохрани все факты, цифры, имена, ссылки <a href="...">, кроме тех,
      что пользователь явно просит изменить.
    - НЕ возвращай весь текст статьи — только заменяющий вариант фрагмента.

  Режим INSERT (selected_text == «— (фрагмент не выделен)»):
    - Верни самодостаточный HTML-блок (1–3 абзаца, или мини-секция с <h3>),
      имеющий смысл вставленного НИЖЕ текущего курсора.
    - НЕ дублируй информацию из full_article_text.
    - 80–400 слов чистого текста, если пользователь не указал иной объём.
    - НЕ начинай с <h2>, если пользователь явно не попросил («H2: …»).

  Запрещено всегда:
    - выводить «вот ваш текст», «конечно», «итак», «ниже исправленный вариант»
      и любые другие вводные;
    - оборачивать ответ в Markdown-fence (```html, ``` и т. п.);
    - возвращать пустой ответ — если запрос пользователя пуст, а фрагмент
      выделен, просто отредактируй стиль фрагмента (исправь ошибки, улучши
      читабельность), сохраняя смысл и разметку.

[SELF-AUDIT — пройди молча перед выводом]
  ✓ Какой режим: REPLACE или INSERT?
  ✓ В режиме REPLACE — ровно ли возвращён только фрагмент, без чужих обёрток?
  ✓ Уложился ли в лимиты по длине?
  ✓ Сохранена ли HTML-структура исходника (REPLACE) / создан валидный блок (INSERT)?
  ✓ Нет ли вводных оборотов и markdown-fence?
  ✓ Ответ непустой?
  Если хотя бы один пункт «нет» — переработай ответ ДО вывода.

[STRICT OUTPUT RULES]
  Выведи ТОЛЬКО result_html — без префиксов, без пояснений, без markdown-обёрток.
  Никогда не возвращай пустой ответ.
