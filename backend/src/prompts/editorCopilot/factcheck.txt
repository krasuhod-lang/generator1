FACTCHECK / Замена факта в выделенном фрагменте (DSPy-Signature).

[INPUTS]
  selected_text       : HTML/Markdown — фрагмент, в котором нужна замена.
  user_instruction    : пара «старое → новое» либо описание факта (например: «Замени Анапу на Геленджик», «Поправь год основания на 2007»).
  full_article_text   : контекст всей статьи (для согласованности).
  lsi_unused          : список LSI-слов (НЕ обязательно использовать в этом действии).

[OUTPUT]
  edited_fragment     : HTML/Markdown — ОБНОВЛЁННЫЙ выделенный фрагмент.
                        Тот же тип разметки, тот же набор тегов, ±15 % по длине.

[REASONING STEPS — выполни молча, в ответе НЕ выводи]
  1) Определи в user_instruction конкретную пару «старый факт → новый факт».
     Если пары нет — мысленно сформулируй её сам, исходя из выделенного фрагмента.
  2) Найди в selected_text ВСЕ упоминания старого факта (включая склонения,
     падежи, единственное/множественное число, синонимы и производные —
     например «Анапа», «Анапе», «Анапы», «анапский»).
  3) Замени каждое упоминание корректной формой нового факта; согласуй
     окружающие прилагательные, причастия и предлоги, чтобы предложение
     осталось грамматически и пунктуационно правильным.
  4) Если меняется число / дата / единица измерения — пересчитай зависимые
     значения в том же предложении (например, год → возраст).
  5) Все остальные предложения, слова и обороты, не связанные с фактом,
     ОСТАВЬ БУКВАЛЬНО НЕИЗМЕННЫМИ. Не «улучшай» стиль, не сокращай, не
     удаляй вводные конструкции, не меняй порядок предложений.

[OUTPUT CONTRACT — обязательные ограничения]
  - Сохрани ту же разметку, что в selected_text: HTML → HTML, Markdown → Markdown.
  - Не добавляй и не удаляй HTML-теги (<p>, <h2>, <h3>, <ul>, <li>, <strong>,
    <em>, <a>); количество и тип открывающих/закрывающих тегов должно совпадать.
  - Длина ответа: 0.85 × len(selected_text) ≤ len(answer) ≤ 1.15 × len(selected_text).
  - Сохрани все цифры, ссылки <a href="...">, имена, термины ниши, кавычки,
    единицы измерения — если они НЕ являются заменяемым фактом.
  - Сохрани LSI-слова, которые уже стояли во фрагменте (не вычёркивай их
    при пересборке предложения).
  - Не вставляй вводных «Конечно», «Вот ваш текст», «Ниже исправленный вариант».

[SELF-AUDIT — пройди молча перед выводом]
  ✓ Все ли формы старого факта заменены? (нашёл ли ты «анапский», а не только «Анапа»)
  ✓ Согласованы ли окружающие слова в роде/числе/падеже?
  ✓ Совпадает ли структура HTML/Markdown с исходником?
  ✓ Ровно ли передан смысл соседних предложений (не относящихся к факту)?
  ✓ Уложился ли в ±15 % по длине?
  Если хотя бы один пункт «нет» — переработай ответ ДО вывода.

[STRICT OUTPUT RULES]
  Выведи ТОЛЬКО edited_fragment — без обёрток ```html```, без комментариев,
  без префиксов, без пояснений. Никогда не возвращай пустой ответ; если
  замена технически невозможна (факт во фрагменте не найден) — верни
  selected_text без изменений.
