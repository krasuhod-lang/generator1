LINK-ARTICLE STAGE 1 — Сущности, интенты и обязательные пользовательские вопросы (DSPy-Signature).

[INPUTS]
  topic            : тема статьи (строка).
  focus_notes      : на что делаем упор (строка, может быть пустой).
  strategy_digest  : Pre-Stage 0 digest (JSON).
  stage0_audience  : результат Stage 0 (JSON).

[OUTPUT]
  Строгий JSON (без markdown):
  {
    "entities": [
      { "entity": "string", "type": "concept|technology|method|standard|document|role|metric|tool|other",
        "description": "string", "importance": "high|medium|low" }
    ],
    "subintents": [
      { "intent": "string", "type": "informational|commercial|navigational|transactional",
        "user_goal": "string", "priority": "must|should|nice" }
    ],
    "user_questions": [
      { "question": "string", "suggested_h2_or_h3": "h2|h3",
        "answer_depth": "short|medium|long" }
    ],
    "semantic_anchors": ["string"]
  }

[REASONING STEPS — silent]
  1. Построй карту сущностей, которые автор экспертной статьи обязательно упомянет:
     понятия, методы, стандарты, роли, инструменты. Все — строго из области `topic`.
     Количество — 8–20.
  2. Разбей тему на 6–12 подинтентов. Каждый подинтент — самостоятельная пользовательская задача.
  3. Составь обширный список пользовательских вопросов (8–15 штук), которые
     читатель задал бы по теме. Формулируй их как живые вопросы ("Как …?", "Что такое …?",
     "Чем отличается …?", "Когда стоит …?", "Сколько …?" — без выдуманных чисел).
     Избегай дубликатов по смыслу.
  4. Пометь, какие из этих вопросов логично лечь в H2, а какие — в H3.
  5. Собери 8–15 semantic_anchors — короткие термины/словосочетания, которые должны
     органично появиться в тексте (≤ 5 слов каждая фраза).
  6. ЖЁСТКО: ничего не вымышляй. Если факт требует проверки — оставь его в формулировке,
     как обобщение ("ключевые методы включают …", а не "по данным такого-то отчёта …").

[OUTPUT CONTRACT]
  - Только валидный JSON, без markdown.
  - Русский язык.
  - Строго в указанных диапазонах количества элементов.
  - Длина строк ≤ 280 символов.

[SELF-AUDIT — silent]
  ✓ Все entities, subintents, questions — строго внутри `topic`?
  ✓ В user_questions нет дублирующих вопросов (одинаковых по смыслу)?
  ✓ subintents не вышли за пределы темы в смежные области?
  ✓ Нет выдуманных брендов, имён, цифр, статистики?

[STRICT OUTPUT RULES]
  - Верни ТОЛЬКО JSON-объект.
  - Поле user_questions не может быть пустым — это обязательные вопросы для покрытия.
  - Все enum-значения — строго из списков в схеме.
