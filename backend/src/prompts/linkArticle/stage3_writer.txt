LINK-ARTICLE STAGE 3 — Написание ссылочной статьи на Gemini 3.1 Pro Preview (DSPy-Signature).

[INPUTS]
  topic            : тема статьи (строка).
  anchor_text      : анкор, который ОБЯЗАН быть в тексте ровно 1 раз (строка).
  anchor_url       : URL ссылки (строка, валидный http/https).
  focus_notes      : на что делаем упор (строка).
  output_format    : 'html' | 'formatted_text' (всегда генерируем HTML; frontend конвертирует).
  stage0_audience  : Stage 0 output (JSON).
  stage1_intents   : Stage 1 output (JSON) — user_questions, entities, subintents, semantic_anchors.
  stage2_structure : Stage 2 output (JSON) — план секций, anchor_plan, image_plan.

[OUTPUT]
  Строгий JSON (без markdown-обёрток):
  {
    "article_html":       "string",
    "self_audit": {
      "anchor_placed":                true,
      "anchor_section_index":         1,
      "anchor_within_first_20pct":    true,
      "all_user_questions_covered":   true,
      "image_slots_present":          [1, 2, 3],
      "hallucination_free":           true,
      "notes":                        "string"
    }
  }

[REASONING STEPS — silent]
  1. Сначала мысленно прочитай весь план (stage2_structure) — H1, все H2/H3, anchor_plan,
     image_plan — и сверь с user_questions из Stage 1. Убедись, что ни один вопрос
     читателя не останется без ответа.
  2. Пиши статью СЕКЦИЮ ЗА СЕКЦИЕЙ, строго в заданном порядке. Сохраняй перечень H2 и H3.
  3. Во ВСТУПЛЕНИИ сразу обозначь пользу материала. Не лей воду. 2–4 абзаца.
  4. В секции с anchor_plan.target_section_index (H2 #1 или #2) органично упомяни
     `anchor_text` ВНУТРИ связного предложения-мысли и оформи как:
         <a href="ANCHOR_URL">ANCHOR_TEXT</a>
     — ровно один раз во всей статье, именно внутри этой секции.
  5. В тех секциях, где stage2_structure.image_plan назначил слот, вставь на своё
     место строчку-плейсхолдер:
         <!-- IMAGE_SLOT_1 -->
         <!-- IMAGE_SLOT_2 -->
         <!-- IMAGE_SLOT_3 -->
     — каждый комментарий строго один раз, ровно в той секции, на которую
     указывает stage2_structure.image_plan[*].target_section_index, между абзацами.
  6. Следи за плотностью смысла: каждый абзац отвечает на конкретный вопрос
     читателя из user_questions. Старайся комбинировать теорию (что такое),
     практику (как), сравнение (чем отличается), ошибки (чего избегать).
  7. Запреты на галлюцинации:
       – Не выдумывай бренды, компании, имена, названия продуктов, URL.
       – Не используй проценты, цифры, стоимости, сроки, если они не могут быть
         объективно известны из общего знания о теме. Если число нужно — переводи
         в обобщение ("в большинстве случаев", "в ряде методов").
       – Не пиши «по данным исследования X», «согласно отчёту Y», «в 2023 году
         рынок вырос на N %» — это запрещённые формулы.
       – Не используй прямые цитаты реальных людей.
       – Не придумывай стандарты и документы, если не уверен (например, «ISO 12345»).
  8. Тон — как задано в stage0_audience.tone_of_voice. Русский язык. Без канцеляризма.

[ARTICLE HTML FORMAT]
  - Статья — чистый HTML-фрагмент (без <html>, <head>, <body>).
  - Структура:
      <h1>…</h1>
      <p>…вступление (2–4 абзаца)…</p>
      <h2>Секция 1</h2>
      <p>…</p>
      <h3>…</h3>   (если нужно)
      <p>…</p>
      <!-- IMAGE_SLOT_N -->   (если в секции назначен слот)
      … и т.д.
      <h2>Заключение</h2>
      <p>…</p>
  - Разрешённые теги: h1, h2, h3, p, ul, ol, li, strong, em, a, blockquote.
  - Запрещено: <script>, <style>, <iframe>, inline onclick/onerror/onload.
  - Для значений атрибута href запрещены data:-URL (разрешены только http/https).
    При этом внутри статьи НЕ нужно вручную вставлять <img> — изображения
    добавляются на бэкенде путём замены плейсхолдеров <!-- IMAGE_SLOT_N -->
    на <figure><img src="data:image/..."><figcaption/></figure>. Это допустимое
    исключение для src-атрибута img и не противоречит запрету data: для href.
  - Все внутренние ссылки (кроме anchor) — только https:// и только если это
    общеизвестные официальные ресурсы. Если сомневаешься — не ставь ссылку.
  - Длина статьи: 900–1800 слов (подбирай по глубине темы).
  - Заголовок <h1> ровно один, он соответствует stage2_structure.h1.

[OUTPUT CONTRACT]
  - Только валидный JSON: `{"article_html": "...", "self_audit": {...}}`.
  - article_html — одна HTML-строка.
  - self_audit.anchor_placed === true и self_audit.image_slots_present равен [1,2,3].
  - Никаких markdown, backtick-обёрток, комментариев снаружи JSON.

[SELF-AUDIT — silent, но ОБЯЗАТЕЛЬНО заполни self_audit в выходе]
  ✓ anchor_text встречается ровно 1 раз внутри <a href="anchor_url">…</a>.
  ✓ Эта <a> находится в H2 №1 или H2 №2 (первые 20 % объёма текста).
  ✓ Все user_questions из Stage 1 закрыты.
  ✓ Плейсхолдеры <!-- IMAGE_SLOT_1 -->, <!-- IMAGE_SLOT_2 -->, <!-- IMAGE_SLOT_3 -->
    присутствуют каждый ровно 1 раз.
  ✓ Нет выдуманных фактов, статистики, брендов, цитат реальных людей.
  ✓ HTML валиден: все теги закрыты, вложенность корректна.

[STRICT OUTPUT RULES]
  - Верни ТОЛЬКО JSON. Никаких префиксов/суффиксов.
  - Запрещено возвращать пустой article_html или заглушку.
  - Запрещено пропустить anchor или image-слоты.
  - При нарушении любого правила пересобери статью до тех пор, пока правило не выполнено.
