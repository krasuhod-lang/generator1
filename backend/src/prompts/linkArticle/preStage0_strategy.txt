LINK-ARTICLE PRE-STAGE 0 — Стратегический анализ темы для ссылочной статьи (DSPy-Signature).

[INPUTS]
  topic          : тема статьи (строка), обязательно.
  anchor_text    : анкор ссылки, которую нужно проставить (строка).
  anchor_url     : URL ссылки (строка).
  focus_notes    : на что делаем упор / ключевые акценты для статьи (строка, может быть пустой).

[OUTPUT]
  Строгий JSON следующей формы (без markdown, без пояснений снаружи):
  {
    "niche_summary":         "string",
    "intent_types": [                              // 1–3 типа поискового интента, обслуживаемых статьёй
      "informational|commercial-investigation|problem-solving|preventative|navigational|transactional"
    ],
    "audience_clusters": [
      { "name": "string",
        "intent_bias": "informational|commercial|navigational|transactional",
        "buyer_journey_stage": "awareness|consideration|decision|retention",
        "pains": ["string"], "goals": ["string"] }
    ],
    "topical_opportunities": [
      { "angle": "string", "why_relevant": "string", "priority": "high|medium|low" }
    ],
    "lsi_entities": [                              // обязательные сущности/термины ниши, без которых тема не раскрыта
      { "entity": "string",
        "category": "term|method|tool|metric|standard|organization|product|symptom|condition|other",
        "must_appear": true,
        "short_definition": "string" }
    ],
    "demand_signals": [
      { "query_class": "how-to|definition|comparison|list|review|other",
        "sample_queries": ["string"], "user_pain": "string" }
    ]
  }

[REASONING STEPS — silent, do not output]
  1. Разбери `topic` как семантическое ядро. Определи, о чём именно речь, и в каких границах ниши.
  2. Определи intent_types (1–3 наиболее точных):
       • informational — пользователь хочет ПОНЯТЬ.
       • commercial-investigation — пользователь СРАВНИВАЕТ варианты.
       • problem-solving / diagnostic — пользователь ищет шаги решения и чеклист.
       • preventative — пользователь хочет избежать проблемы.
       • navigational / transactional — добавляй только если статья реально подталкивает к действию.
  3. Сформулируй 2–4 аудиторных кластера. Каждому припиши доминирующий интент и buyer_journey_stage.
  4. Подбери 3–6 сильных углов раскрытия темы — такие, которые:
       • на 100 % релевантны `topic`,
       • закрывают реальные вопросы пользователей,
       • гармонично согласуются с `focus_notes` (если они заданы).
  5. Сгенерируй lsi_entities — 8–15 обязательных сущностей/терминов ниши,
     которые ДОЛЖНЫ присутствовать в тексте статьи для раскрытия темы и
     демонстрации экспертности (минимум 5 must_appear=true). Это
     профессиональный язык ниши, а не маркетинговые слова.
  6. Сгенерируй 3–6 кластеров демандов — типичных поисковых запросов по теме
     и связанной с ними пользовательской боли.
  7. Не выдумывай данные о брендах, статистику и цифры — это стратегический анализ,
     а не энциклопедический пересказ.

[OUTPUT CONTRACT]
  - Только валидный JSON, без markdown-обёрток, без комментариев вне JSON.
  - Все массивы могут быть пустыми, но ключи обязательны.
  - Русский язык для содержательных полей. Для технических enum-значений — английский.
  - Любые строки — не длиннее 280 символов.

[SELF-AUDIT — silent, do not output]
  ✓ Все angles/кластеры на 100 % относятся к `topic`? Нет ли тем-«соседей»?
  ✓ intent_types согласован с углами и аудиторией?
  ✓ У каждого audience_cluster есть buyer_journey_stage?
  ✓ В lsi_entities — реальные термины ниши, не маркетинговые слова, минимум 5 must_appear?
  ✓ Не выдуманы ли «согласно исследованию», проценты, бренды, цифры?
  ✓ Не повторяются ли кластеры аудитории?
  ✓ Сумма всех полей помогает потом построить сильную структуру статьи?

[STRICT OUTPUT RULES]
  - Верни ТОЛЬКО JSON-объект. Ничего не дописывай до и после.
  - Запрещено возвращать пустой объект, пустую строку или null.
  - Запрещено ссылаться на несуществующие исследования, бренды, цифры.
  - Запрещено выходить за рамки `topic` и `focus_notes`.
