LINK-ARTICLE STAGE 2 — Структура ссылочной статьи с плейсхолдерами для анкора и изображений (DSPy-Signature).

[INPUTS]
  topic               : тема статьи (строка).
  anchor_text         : анкор, который ОБЯЗАН быть в тексте (строка).
  anchor_url          : URL ссылки (строка).
  focus_notes         : на что делаем упор (строка).
  stage0_audience     : Stage 0 output (JSON).
  stage1_intents      : Stage 1 output (JSON) — обязательный набор user_questions.
  whitespace_hints    : Stage 1B output → article_hierarchy_hints (JSON, обязательный):
                        { must_cover_subtopics, must_cover_intents,
                          must_cover_audiences, preferred_formats, h2_ideas }.

[OUTPUT]
  Строгий JSON (без markdown):
  {
    "h1":                   "string",
    "intro_brief":          "string",
    "sections": [
      {
        "index":                   1,
        "h2":                      "string",
        "descriptor":              "string",
        "covers_user_questions":   ["string"],
        "covers_subtopics":        ["string"],     // подмножество whitespace_hints.must_cover_subtopics
        "covers_intent":           "informational|commercial|navigational|transactional",
        "must_include_entities":   ["string"],
        "subsections":             [{ "h3": "string", "descriptor": "string" }],
        "anchor_insertion":        false,
        "image_slot":              null
      }
    ],
    "anchor_plan": {
      "anchor_text":          "string",
      "anchor_url":           "string",
      "target_section_index": 1,
      "natural_context":      "string"
    },
    "image_plan": [
      {
        "slot":                  1,
        "target_section_index":  0,
        "what_to_visualize":     "string",   // 1 предложение, по делу
        "scene_concept":         "string",   // 1–2 предложения: ЖИВАЯ сцена (не абстракция)
        "subject_focus":         "string"    // КТО / ЧТО в центре кадра (человек+действие или предмет+контекст)
      },
      { "slot": 2, "target_section_index": 0, "what_to_visualize": "string", "scene_concept": "string", "subject_focus": "string" },

    "hierarchy_audit": {
      "covered_subtopics":  ["string"],   // какие must_cover_subtopics были разложены по секциям

    }
  }

[REASONING STEPS — silent]
  1. Придумай H1 (≤ 90 символов), который полностью отражает `topic`. Не перегружай ключами.
  2. Возьми `whitespace_hints.h2_ideas` как стартовый набор H2-кандидатов и
     синтезируй 5–8 H2-секций так, чтобы вместе они закрывали:
        • ВСЕ `user_questions` из Stage 1;
        • КАЖДЫЙ `must_cover_subtopics` из whitespace_hints;
        • ВСЕ `must_cover_intents` (разнеси их по подходящим секциям).
     Если белые пятна (whitespace_hints.must_cover_subtopics) и user_questions
     указывают в одну сторону — приоритизируй ту трактовку, где сильнее
     уникальный угол (white-space win), а не очевидный keyword-research-список.
  3. Для каждой секции укажи `covers_subtopics` (подмножество must_cover_subtopics)
     и `covers_intent` (один из must_cover_intents).
  4. Назначь H3-подразделы там, где это объективно нужно (по 0–3 на H2).
  5. ЖЁСТКОЕ ПРАВИЛО (anchor):
       – Выбери target_section_index = 1 или 2 (то есть H2 №1 либо №2).
       – Только у этого раздела anchor_insertion = true. У остальных — false.
       – natural_context объясняет, в какую смысловую мысль встроить `anchor_text`
         так, чтобы ссылка выглядела естественно, а не «приклеенно».
  6. ЖЁСТКОЕ ПРАВИЛО (images): ровно 3 image-слота в трёх РАЗНЫХ секциях.
        • `what_to_visualize` — что полезно показать (одно предложение).
        • `scene_concept` — ЖИВАЯ сценарная подсказка (1–2 предложения):
           конкретное действие конкретного субъекта в конкретном окружении.
           Думай как фоторепортёр / документалист, не как иконографик.
        • `subject_focus` — КТО (человек + действие) или ЧТО (предмет + контекст)
           в фокусе. Категорически избегай абстракций «диаграмма / схема /
           инфографика / 3D-render». Никаких реальных брендов и знаменитостей.
  7. В `hierarchy_audit` зафиксируй, какие subtopics и intents из whitespace_hints
     ты реально разложил по секциям — это контрольный чек.
  8. Ничего не выдумывай про бренды, статистику, реальные компании, документы.
  9. Не выходи за рамки `topic` и `focus_notes`.

[OUTPUT CONTRACT]
  - Только валидный JSON, без markdown.
  - Русский язык в содержательных полях.
  - sections: 5–8 элементов.
  - В sections: ровно один элемент с anchor_insertion=true, и его index ∈ {1, 2}.
  - В image_plan: ровно 3 элемента с slot = 1, 2, 3.
  - `image_plan[*].target_section_index` — валидный index из sections, и три слота
     ссылаются на ТРИ РАЗНЫЕ секции.
  - anchor_plan.target_section_index совпадает с index того раздела, у которого
     anchor_insertion = true.
  - `hierarchy_audit.covered_subtopics` покрывает ≥ 80 % must_cover_subtopics
     (любой пропуск — прямой нарушенный SLA).

[SELF-AUDIT — silent]
  ✓ Все user_questions из Stage 1 присутствуют хотя бы в одном covers_user_questions?
  ✓ Все must_cover_subtopics из whitespace_hints отражены в covers_subtopics?
  ✓ anchor_insertion ровно у одной секции, и её index = 1 или 2?
  ✓ image_plan — ровно 3 слота в 3 разных секциях?
  ✓ Каждое scene_concept описывает ЖИВУЮ сцену (есть субъект + действие/контекст,
    не «инфографика», не «3D-визуализация», не «диаграмма»)?
  ✓ Нет дублей H2 / H3 по смыслу?
  ✓ Нет выдуманных фактов?

[STRICT OUTPUT RULES]
  - Верни ТОЛЬКО JSON-объект.
  - Любое нарушение правил anchor/image/whitespace — фатально, пересобери структуру.
  - Не добавляй полей сверх схемы.
