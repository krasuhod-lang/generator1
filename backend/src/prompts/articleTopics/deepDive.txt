## 1. Семантическое ядро будущего

Сгенерируй 30–50 запросов, которые появятся когда тренд войдёт
в стадию массового принятия. Разбей по интенту:

- Информационные (что это, как работает)
- Коммерческие (купить, заказать, цена, сравнение)
- Навигационные (бренды, инструменты, платформы)
- Проблемные (не работает, ошибка, альтернатива)

Для каждой группы — markdown-список с короткими пояснениями
(1–2 запроса с пометкой why).

## 2. Контентная архитектура Hub & Spoke

Нарисуй структуру: 1 Pillar Page + 5–7 Cluster Pages + перелинковочная
логика между ними. Для каждой страницы — таблица:

| Тип (Pillar / Cluster) | URL-slug | H1 | Meta description (≤155) | Основные H2 (3–6) | На какие cluster-страницы ссылается |

## 3. Конкурентный gap

Что из этой структуры НЕ покрыто топ-10 конкурентов прямо сейчас?
Какие страницы создать первыми (приоритет 1–3)? Кратко обоснуй каждый
приоритет (1–2 строки).

## 4. Быстрая победа

Одна страница, которую можно опубликовать на этой неделе, чтобы
застолбить нишу. Полная структура этой страницы:

- H1 (≤70 символов)
- Meta title (50–60 символов)
- Meta description (140–155 символов)
- Slug
- Список H2/H3 с короткими аннотациями (1–2 строки на каждый)
- 3–5 ключевых тезисов или статистик, которые должны быть в тексте
- 1 CTA-блок: что предлагаем читателю и куда ведёт ссылка
