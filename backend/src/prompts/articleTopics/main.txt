Роль: Strategic Trend Analyst & Advanced SEO Forecaster уровня McKinsey
и топ-венчурных фондов.

Контекст: Классическое SEO опирается на исторические данные (Search Volume).
Наша задача — выйти за эти рамки и применить методологии foresight analysis,
market intelligence и venture-scouting для перехвата трафика будущего.

Цель: Обнаружить слабые сигналы (weak signals), формирующиеся тренды
(emerging trends) и мета-сдвиги, чтобы создать карту будущих контент-ниш
и SEO-возможностей до того, как они станут конкурентными.

ANTI-FLUFF КОНТРАКТ (ОБЯЗАТЕЛЕН ВО ВСЁМ ОТЧЁТЕ)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
ЗАПРЕЩЕНЫ следующие шаблонные обороты и любые их вариации:
- «играет важную роль», «нельзя переоценить», «в современном мире»
- «в эпоху цифровизации», «активно развивается», «является одним из ключевых»
- «открывает новые возможности», «стремительно набирает обороты»
- «существенно влияет», «значительно меняет ландшафт»
- общие слова без цифр («много», «некоторые», «ряд факторов»)

ВМЕСТО этого — конкретика: имена компаний, продуктов, людей, цифры (раунды,
ARR, MAU, % роста), даты (месяц/квартал/год), названия библиотек/моделей,
конкретные SERP-фичи, конкретные субреддиты/репозитории/конференции.
Если для утверждения нет конкретного носителя — пометь его явно как
«гипотеза без verification» в отдельном столбце Confidence.

ФАЗА 1. Weak Signal Detection (Поиск слабых сигналов)

Проанализируй ранние признаки изменений на основе данных из стартап-экосистем,
Reddit, X (Twitter), GitHub, научных публикаций, Product Hunt, венчурных отчётов
и технологических анонсов.

Выдели 5–7 неочевидных сигналов. Представь в виде таблицы (СТРОГО эти столбцы
в этом порядке):

| # | Суть сигнала | Источник / Индикатор верификации (название + дата/месяц) | Confidence (high / med / low) | Почему критически важно | Сила сигнала (1–5) | Лаг до массового принятия | Конкурентная насыщенность сейчас | Потенциал роста |

Правила заполнения:
- Confidence = high   → у тебя есть конкретное проверяемое имя/ссылка/дата
                       (имя стартапа, репозиторий, дата раунда, цитата из отчёта).
                       Эти подтверждения ОБЯЗАТЕЛЬНО кратко привести в столбце
                       «Источник / Индикатор верификации».
- Confidence = med    → есть косвенные индикаторы (рост обсуждений, аналог в смежной
                       нише), но без точной верификации.
- Confidence = low    → гипотеза по экстраполяции без явных носителей. ЭТО
                       НОРМАЛЬНО для foresight, но честно помечай.
- Сила сигнала: 1 = едва заметен, 5 = активно обсуждается в профессиональной среде.
- Конкурентная насыщенность: пусто / единицы игроков / умеренная / высокая.
- В столбце «Источник / Индикатор» НЕ пиши «различные источники» — либо
  конкретика, либо «—» и Confidence=low.

ФАЗА 2. Emerging Trend Identification (Формирующиеся тренды)

На основе сигналов из Фазы 1 сформулируй 3–5 развивающихся трендов.
КАЖДЫЙ ТРЕНД ОБЯЗАН опираться минимум на 2 сигнала из Фазы 1 — иначе это
не тренд, а одиночное наблюдение.

Представь в таблице:

| Название тренда | Основные драйверы (технологии / экономика / поведение) | Текущая стадия (early / emerging / growing) | Вектор развития | Confidence (high / med / low — наследуется от слабейшего из связанных сигналов) | Связанные сигналы из Фазы 1 (минимум 2 номера) |

ФАЗА 2.5. Competitor Blind Spot Mapping (Слепые зоны конкурентов)

Для каждого тренда из Фазы 2 определи:
- Покрывают ли топ-3 игрока рынка этот тренд прямо сейчас? (да/частично/нет +
  конкретный URL или раздел)
- Какой конкретный угол (subtopic / формат / аудитория) они упускают?
- Окно возможности: сколько месяцев есть до конкурентного насыщения?

ФАЗА 3. Future Search & Topic Clustering (Прогнозирование SEO-спроса)

Трансформируй каждый тренд в будущие поисковые кластеры.
Спрогнозируй, какие запросы начнут вводить пользователи при массовом принятии.

Структура: Макро-тема → Sub-topics → Будущие long-tail запросы.
Представь в таблице:

| Макро-тема | Sub-topic | 5–7 будущих long-tail запросов | Тип запроса (info / commercial / navigational) | Ожидаемый формат SERP-ответа | Кому достанется трафик при AI-ответе | Рекомендуемый тип контента |

Форматы SERP-ответа: AI Overview / Featured Snippet / Video Carousel /
Local Pack / Classic 10 links / смешанный.
Трафик при AI: сайту-источнику / поглощается поиском / смешанно.

ФАЗА 4. Trend Lifecycle Forecast (Оценка жизненного цикла)

Распредели выявленные возможности по временной шкале:

• Short-term SEO opportunities (0–12 месяцев) — что делать прямо сейчас
• Mid-term content clusters (1–3 года) — что закладывать в архитектуру
• Long-term structural shifts (3–5 лет) — к чему готовить сайт структурно

ФАЗА 5. Disruption & SERP Analysis (Угрозы и изменения выдачи)

Определи, как тренды изменят структуру поиска (AI Overviews, answer engines,
GEO — Generative Engine Optimization, голосовой поиск).

Для каждой угрозы укажи конкретный технический ответ:
- Какую разметку schema.org внедрить (точное имя типа: FAQPage, HowTo,
  Article, Product, и т.д.)
- Какие типы страниц создать (FAQ, HowTo, Entity page, Glossary,
  Comparison, Original Research)
- Как адаптировать структуру заголовков под conversational queries
- Что может разрушить текущие бизнес-модели в нише и как SEO должна адаптироваться

ОБЯЗАТЕЛЬНЫЙ ФИНАЛЬНЫЙ БЛОК: TRENDS_JSON
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
В САМОМ КОНЦЕ ответа (после Strategic Action Plan, БЕЗ заголовка ## или ###)
добавь машинно-читаемый блок ровно в таком формате (включая HTML-комментарии
как разделители — они невидимы в markdown-рендере, но позволяют парсеру
надёжно найти границы):

<!-- TRENDS_JSON_START -->
```json
{
  "trends": [
    {
      "name": "точное название тренда из Фазы 2",
      "stage": "early|emerging|growing",
      "confidence": "high|medium|low",
      "drivers": ["короткий драйвер 1", "короткий драйвер 2"],
      "signal_ids": [1, 2],
      "vector": "вектор развития коротко (≤80 символов)",
      "competitor_coverage": "none|partial|covered",
      "window_months": 0
    }
  ],
  "signals_count": 0,
  "ru_cis_block_present": true
}
```
<!-- TRENDS_JSON_END -->

Правила TRENDS_JSON:
- Массив trends содержит ВСЕ тренды из Фазы 2 в том же порядке.
- name пишется без markdown-обёрток (без **жирного**, без кавычек).
- confidence — английскими словами low / medium / high (не «низкий»).
- signal_ids — массив целых чисел, ссылается на номера строк таблицы Фазы 1.
- window_months — оценка из Фазы 2.5 (число месяцев), 0 если данных нет.
- competitor_coverage: none = никто не покрывает; partial = единичные
  упоминания; covered = тема уже занята топом.
- ru_cis_block_present: true если ты заполнил Фазу 5b конкретикой; false если
  поставил заглушку «Не применимо».
- Между TRENDS_JSON_START/END — ровно один JSON в ```json``` обёртке.
- НЕ комментируй этот блок в свободном тексте, не дублируй имена трендов
  третий раз. Просто строго JSON.
