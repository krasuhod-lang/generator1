# Подключение Яндекс.Вебмастера — пошаговая инструкция

> Модуль **«Проекты»** связывает SEO-проект с вашим аккаунтом **Яндекс.Вебмастер**
> по протоколу **OAuth 2.0** (Яндекс ID) и подтягивает статистику поисковых
> запросов (показы, клики, средняя позиция, CTR) для дашборда и сопоставления
> с Google Search Console.
>
> Интеграция реализована **симметрично** Google Search Console (см.
> `GSC_SETUP.md`). OAuth-токены хранятся в БД **строго в зашифрованном виде**
> (AES-256-GCM).

---

## TL;DR

1. В **Яндекс OAuth** (https://oauth.yandex.ru/client/new) создайте приложение
   и выдайте ему доступ к **Яндекс.Вебмастеру**.
2. Пропишите 3 секрета окружения: `YANDEX_CLIENT_ID`, `YANDEX_CLIENT_SECRET`,
   `YANDEX_OAUTH_REDIRECT_URI`.
3. Перезапустите backend.
4. В интерфейсе: **Проекты → открыть проект → вкладка «Яндекс.Вебмастер» →
   «Подключить Яндекс.Вебмастер» → выбрать сайт**.

---

## Часть 1. Настройка на стороне Яндекса

1. Откройте [Яндекс OAuth](https://oauth.yandex.ru/) и создайте приложение
   («Создать приложение»).
2. Платформа — **Веб-сервисы**. В поле **Redirect URI** укажите адрес колбэка,
   например `https://<ваш-домен>/api/oauth/yandex/callback`
   (канонический путь — `https://<ваш-домен>/api/public/projects/ydx/callback`).
3. В разделе **Доступы** выдайте права Яндекс.Вебмастера:
   - `webmaster:hostinfo` — чтение информации о сайтах и статистики;
   - `webmaster:verify` — список подтверждённых сайтов.
4. Сохраните приложение и скопируйте **ClientID** и **Client secret**.

---

## Часть 2. Переменные окружения

| Переменная | Обяз. | Значение |
| --- | --- | --- |
| `YANDEX_CLIENT_ID` | да | ClientID приложения Яндекс OAuth |
| `YANDEX_CLIENT_SECRET` | да | Client secret приложения |
| `YANDEX_OAUTH_REDIRECT_URI` | да | Точный Redirect URI, заданный в приложении |

> Значения читаются из окружения по месту (файл `.env.example` не меняется).
> Если переменные не заданы — кнопка подключения во вкладке «Яндекс.Вебмастер»
> остаётся неактивной, остальной функционал проектов работает.

---

## Часть 3. Что появляется в интерфейсе

- В карточке проекта — **три вкладки**: «Google Search Console»,
  «Яндекс.Вебмастер» и «Сравнение и рекомендации».
- Во вкладке «Яндекс.Вебмастер» — подключение по OAuth, выбор сайта и график
  эффективности с **раздельными** метриками (как в нативном GSC): «Показы и
  клики», «Позиция» (посуточная средняя) и «CTR».
- Во вкладке «Сравнение и рекомендации» — сводная таблица показателей обеих
  систем, пересечение поисковых запросов (общие / только Google / только
  Яндекс) и автоматические рекомендации по улучшению видимости.

> Поисковые запросы Яндекс.Вебмастера тянутся **без лимита** — постранично
> выбираются все запросы за период (см. `projects/ydxClient.js` →
> `queryPopularAll`, настройки `ydx.pageSize` / `ydx.maxRows` в `config.js`).

## Коды ошибок

| Код | Причина |
| --- | --- |
| `ydx_not_configured` (503) | Не заданы `YANDEX_CLIENT_ID` / `YANDEX_CLIENT_SECRET` / `YANDEX_OAUTH_REDIRECT_URI` |
| `ydx_not_connected` (409) | Проект не подключён к Яндекс.Вебмастеру |
| `ydx_no_host` (409) | Не выбран сайт Яндекс.Вебмастера |
| `ydx_token_refresh_failed` | Не удалось обновить access-токен по refresh |
