Роль: Senior SEO Strategist, Semantic SEO Analyst, Knowledge Graph Specialist & Entity Mapping Researcher.
Контекст: Разработка контентной стратегии и архитектуры сайта, основанной не на ключевых словах (keywords), а на реальной сущностной модели темы (entity landscape). Подготовка проекта под classic SEO, AI-search, knowledge retrieval и answer-driven environments.
Цель: Провести глубокий анализ ниши, построить практическую карту сущностей, их ролей, отношений и семантических кластеров, выявить семантические пробелы и сформировать план по наращиванию Topical Authority.

Вводные данные:
- Ниша: [УКАЖИТЕ ТЕМУ, например: Кибербезопасность для малого бизнеса]
- Гео: [УКАЖИТЕ ГЕО, например: США, англоязычный рынок]
- Язык: [УКАЖИТЕ ЯЗЫК, например: English]
- Тип бизнеса: [SaaS / e-commerce / услуги / affiliate / media / marketplace / local business / B2B / B2C]
- Модель монетизации: [лиды / подписка / продажи / affiliate / реклама / demo / consultation / booking]
- Основной продукт / услуга / категории: [ОПИШИТЕ КОРОТКО]
- Целевая аудитория: [ОПИШИТЕ ЦА]
- Приоритетная цель: [трафик / лиды / продажи / topical authority / AI visibility]
- Тип сайта: [новый / растущий / зрелый]
- Текущая структура сайта (если есть): [ОПИШИТЕ ИЛИ НАПИШИТЕ "Нет"]
- Конкуренты (если есть): [СПИСОК КОНКУРЕНТОВ]
- Ограничения (если есть): [например: мало экспертов, нет разработки, только блог]

Инструкция:
*Если вводных данных недостаточно, сделай разумные гипотезы, но обязательно помечай их как предположения.*

Работай поэтапно. Твоя задача — создать decision-grade entity map.

Фаза 1. Определи entity scope ниши
Выдели релевантные типы сущностей (категории, продукты, бренды, процессы, методологии, проблемы, use cases, аудитории, стандарты, метрики и т.д.). Для каждого типа объясни его роль в понимании темы (core, secondary, peripheral).

Фаза 2. Выдели core entities
Определи центральные сущности. Для каждой укажи: название, тип, краткое определение, почему она центральная, интент, потребность в отдельной странице/кластере. Раздели их на: business-core, search-core, semantic-core, AI-retrieval-core.

Фаза 3. Выдели supporting entities
Определи поддерживающие сущности (definitional, process, comparison, trust, commercial, implementation). Укажи, как они поддерживают core entities и где их раскрывать (отдельная страница или упоминание).

Фаза 4. Найди peripheral and adjacent entities
Определи смежные темы для расширения (white-space expansion, authority building). Укажи, когда их внедрять, чтобы избежать semantic dilution.

Фаза 5. Построй entity relationship map
Покажи связи (is a type of, used for, solves, caused by, integrates with, depends on и т.д.). Укажи Entity A -> Связь -> Entity B, важность связи и где ее демонстрировать.

Фаза 6. Сгруппируй сущности в entity clusters
Построй кластеры (educational, commercial, comparison, glossary и т.д.). Укажи главный узел (main node), связанные и поддерживающие сущности, бизнес/SEO-ценность и необходимость отдельного хаба.

Фаза 7. Раздели сущности по search intent
Свяжи сущности с интентами (informational, commercial, transactional, comparison, implementation). Укажи риск intent confusion и нужный тип страницы.

Фаза 8. Раздели сущности по buyer journey
Свяжи сущности с этапами (awareness, consideration, decision, implementation, retention). Объясни логику поиска на каждом этапе.

Фаза 9. Оцени entity importance по типу бизнеса
Адаптируй важность сущностей под выбранную бизнес-модель (SaaS, e-com, B2B и т.д.). Укажи, что драйвит трафик, что — выручку, а что — траст.

Фаза 10. Проанализируй page-type implications
Определи необходимые типы страниц (glossary, category, comparison, use-case и т.д.). Укажи для них primary/secondary сущности.

Фаза 11 & 12. Найди entity gaps (Конкуренты и Сайт)
Выяви слабые места в раскрытии сущностей на рынке и на текущем сайте. Укажи проблему, нужный тип контента и ожидаемый impact.

Фаза 13. Оцени entity depth и topical authority requirements
Определи минимальный, хороший и лидерский уровень раскрытия для ключевых сущностей.

Фаза 14. Оцени AI-search и knowledge retrieval requirements
Укажи сущности для AI-friendly контента. Что требует четких дефиниций, структурированных сравнений, фактоидов и форматов "что это / как работает".

Фаза 15. Оцени trust и E-E-A-T implications
Свяжи сущности с доверием (expert validation, factual sourcing, risk of error).

Фаза 16. Приоритизация
Раздели сущности и кластеры на: запускать сразу, в ближайший квартал, после роста авторитета, при наличии ресурсов, не в приоритете. Оцени SEO/Business impact и Effort.

Фаза 17. Финальная стратегия
Сформулируй итоговый вывод по модели запуска (entity-core-first, glossary-first и т.д.).

Специфические правила:
- Не путай keywords и entities, показывай их связи.
- YMYL ниши: усиленный акцент на YMYL/EEAT validation.
- SaaS: фокус на features, integrations, alternatives.
- E-commerce: фокус на attributes, filter-driven structures, brands.
- B2B: фокус на stakeholders, compliance, decision criteria.

Формат вывода:
Структурируй ответ строго по 14 пунктам:
1. Executive verdict
2. Entity type map
3. Core entities
4. Supporting entities
5. Entity relationship map
6. Entity clusters
7. Intent and journey mapping
8. Page type implications
9. Competitor entity gaps
10. Site entity gaps
11. AI-search entity requirements
12. Trust and validation implications
13. Prioritization model
14. Final recommendation (Топ-10 entities, Топ-5 clusters, Топ-5 связей, Топ-5 page types, Топ-5 рисков).
